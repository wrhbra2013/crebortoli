Sistema de Agendamento - Espaço Cre Bortoli

Visão Geral

Sistema de agendamento de serviços de beleza com pagamento via PIX. Desenvolvido com HTML, CSS e JavaScript (ES6+), utilizando PHP com SQLite para persistência de dados.

Versão: 1.0.0

URLs do Sistema

Site Principal https://www.crebortoli.com.br/
Agenda (Público) https://www.crebortoli.com.br/paginas/agenda.html
Tabela de Preços https://www.crebortoli.com.br/paginas/servicos.html
Painel Admin (SIG) https://www.crebortoli.com.br/sig/

Fluxo do Cliente

1

Acesso ao Site

Cliente acessa a página de agenda e visualiza o calendário do mês atual. A lista "Meus Agendamentos" mostra os agendamentos já realizados.

2

Fazer Agendamento

Cliente clica em um dia disponível → Abre modal com formulário (nome, WhatsApp, serviço) → Clica em "Confirmar Agendamento" → Dados salvos no localStorage + sync com servidor → Abre WhatsApp com mensagem pré-preenchida.

3

Pagamento via PIX

Cliente clica em "Pagar PIX" na lista → Abre modal com chave PIX e valor → Copia chave ou escaneia QR Code → Após pagar, clica em "Já fiz o pagamento" → Status atualizado no sistema.

4

Visualização

Calendário mostra badges coloridos: Verde = pagos, Vermelho = pendentes.

Fluxo do Administrador (SIG)

1. Acesso ao Painel

URL: sig/ - Dashboard com múltiplas opções: Agenda, Tabela de Preços, Pagamentos, Receituário, Relatórios.

2. Gestão de Agendamentos

Visualização semanal ou mensal. Agendamentos mostrados com badges coloridos:

  • Rosa Pendente
  • Verde Pago/Confirmado
  • Azul Concluído

3. Tabela de Preços

Modo edição permite alterar preços dos serviços. Sincronização via servicos.json no servidor.

4. Gestão de Pagamentos

Lista de agendamentos pendentes, dados PIX, confirmar pagamento, link WhatsApp.

5. Módulos Adicionais

  • Receituário: Controle de produtos e receitas
  • Relatórios: Dados do MEI, vendas, custos
  • Aprovar/Confirmar: Aprovação de agendamentos pendentes

Arquitetura do Sistema

Backend

PHP + SQLite - Servidor web com banco de dados leve:

  • PHP 7.4+ com extensão PDO
  • SQLite (arquivo local)
  • API REST em api.php

Banco de Dados

Tabelas disponíveis no SQLite:

Tabela Descrição
agendamentosAgendamentos de serviços
servicosCatálogo de serviços disponíveis
clientesCadastro de clientes
receitasReceitas de produtos
qr_sessoesSessões de autenticação QR

API REST

API PHP em api.php com SQLite:

GET  api.php?action=agendamentos  # Listar agendamentos
POST api.php                      # Criar/atualizar dados

Autenticação QR

O SIG usa autenticação via QR Code com backend PHP/SQLite:

  • Tabela qr_sessoes no SQLite armazena sessões
  • Usuário gera QR no computador
  • Celular escaneia e aprova com código

Arquivo servicos.json

Arquivo estático para sincronização de preços:

{
    "servicos": [
        { "id": "dep_perna", "nome": "Depilação Perna", "preco": 25, "categoria": "Depilação" }
    ],
    "ultimaAtualizacao": "2026-03-23"
}
