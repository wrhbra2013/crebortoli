Arquitetura do Sistema

Sistema de Agendamento - Espaço Cre Bortoli

Visão Geral

Arquitetura: Frontend + API PHP + SQLite
Persistência: localStorage + Servidor PHP (InfinityFree)
Database: SQLite no servidor

Cenários de Uso

Cenário 1: Sem Internet (Apenas localStorage)

Browser
→
localStorage ✓

✓ Funciona offline
✗ Dados não sincronizam entre dispositivos

Cenário 2: Com Internet (PHP + SQLite)

Browser
↔
localStorage
←→
PHP + SQLite

✓ Dados sincronizam entre dispositivos
✓ Leitura e escrita
✓ Persistência completa

Comportamento dos Stores

MétodolocalStoragePHP + SQLite
getAll()✓ Lê do cache local✓ Sincroniza no init
save(item)✓ Salva localmente✓ Escreve no servidor
update(id, dados)✓ Atualiza localmente✓ Atualiza no servidor
delete(id)✓ Remove localmente✓ Remove do servidor
