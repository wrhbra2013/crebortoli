Deploy no InfinityFree

Resumo: Guia para fazer upload do projeto Cre Bortoli no InfinityFree com PHP + SQLite.

Pré-requisitos

  • Conta no InfinityFree
  • Domínio gratuito (seusite.epizy.com)
  • Arquivos do projeto

Passo a Passo

1. Criar Conta InfinityFree

  1. Acesse infinityfree.net
  2. Clique em Sign Up Free
  3. Escolha um subdomínio (ex: crebortoli.epizy.com)
  4. Complete o registro

2. Fazer Upload apenas do api.php

  1. No painel, vá em Files → Online File Manager
  2. Navegue até htdocs
  3. Faça upload apenas do arquivo api.php
  4. Clique em Upload → selecione o arquivo → Upload Files
Apenas o api.php é necessário no InfinityFree!
Os outros arquivos ficam no seu servidor local ou onde você hospeda o site.

3. Testar a API

Após o upload, teste:

https://SEU_DOMINIO.epizy.com/api.php?action=status

Deve retornar:

{"status":"ok","database":"created","tables":["agendamentos","servicos","clientes","receitas","contatos","vendas_servicos"]}

4. Configurar o JavaScript

Edite static/js/storage.js e substitua a URL pela sua:

const API_URL = 'https://SEU_DOMINIO.epizy.com/api.php';
Importante: Substitua SEU_DOMINIO pelo seu domínio real do InfinityFree!

Testar o Sistema

Após configurar:
  1. Abra seu site local ou no seu servidor
  2. Abra o console (F12) → aba Console
  3. Você deve ver: "Sincronizado com servidor PHP"
  4. Teste fazer um agendamento

Solução de Problemas

Erro 403 ou 500

Verifique se o api.php tem permissões corretas. No File Manager:

  1. Clique em api.php
  2. Clique em Permissions
  3. Marque todas as permissões (ou 644)

API não responde

Verifique se o PHP está ativado:

  1. No painel, vá em PHP
  2. Verifique se há versão PHP ativa
  3. Se não houver, ative uma versão (recomendado: PHP 8.1)

SQLite não funciona

O InfinityFree tem SQLite ativado por padrão. Se der erro:

  1. Verifique se o banco database.sqlite foi criado
  2. No File Manager, procure por database.sqlite na pasta htdocs
  3. Se não existir, a API vai criá-lo automaticamente no primeiro acesso

Dica de Segurança

Após funcionar, você pode restringir o acesso à API:

  • Alterar o nome do arquivo api.php para algo menos óbvio
  • Adicionar autenticação simples
